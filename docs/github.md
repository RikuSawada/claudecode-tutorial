# GitHub 連携

---

## 事前準備

GitHub CLI (`gh`) のインストールと認証が必要です。

```bash
# インストール（macOS）
brew install gh

# 認証
gh auth login
```

認証後は、Claude Code から自然言語で GitHub を操作できます。

---

## Issue の操作

### 一覧を確認する

```
「未解決のIssueを一覧表示して」
「自分にアサインされているIssueを見せて」
```

### 内容を読む

```
「Issue #42 の内容を確認して」
```

### Issue を作成する

```
「ログイン時にパスワードが表示されてしまうバグのIssueを作成して」
```

### Issue を元に実装する

```
「Issue #42 を読んで、修正を実装して」
```

---

## Pull Request の操作

### PR を作成する

```
「この変更でPRを作成して」
「feature/add-login ブランチから main へのPRを作って」
```

### PR をレビューする

```
/review-pr 123
「PR #123 のコードをレビューして」
```

### レビューコメントに対応する

```
「PR #123 のレビューコメントをすべて読んで、対応して」
```

---

## 典型的な開発フロー

```
1. Issueを確認する
   「Issue #10 の内容を読んで」

2. ブランチを作成する
   「feature/fix-login-bug ブランチを作って切り替えて」

3. 実装する
   「Issue の内容に従って修正を実装して」

4. コミットする
   /commit

5. PRを作成する
   「PRを作成して。タイトルと説明はIssue #10 の内容を元に書いて」

6. レビューに対応する
   「PR #20 のレビューコメントに対応して」
```

---

## リリース・履歴の確認

```
「前回のリリースから今日までの変更をまとめて」
「v1.2.0 と v1.3.0 の差分を説明して」
「最近のコミット履歴を要約して」
```

---

## 注意点

Claude Code は以下の操作の前に確認を求めます：

- PR のマージ
- `git push --force`
- ブランチの削除
- Issue / PR のクローズ

意図しない操作を防ぐため、内容をよく確認してから承認してください。
