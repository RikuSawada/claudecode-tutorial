# 基本的な使い方

---

## 会話形式でタスクを依頼する

Claude Code は自然な日本語でタスクを依頼するだけで動作します。
コマンドを覚える必要はありません。

```
「このファイルの内容を説明して」
「このエラーを修正して」
「ユーザー認証機能を追加して」
```

---

## よくあるユースケース

### コードを理解する

```
「src/auth.ts の処理の流れを説明して」
「この関数は何をしているの？」
「このプロジェクトの全体構成を把握したい」
```

### バグを修正する

```
「このエラーメッセージの原因を調べて修正して」
「テストが失敗している。原因を特定して直して」
```

エラーメッセージは直接貼り付けると効果的です：

```
以下のエラーを修正して：

TypeError: Cannot read properties of undefined (reading 'map')
  at UserList (src/components/UserList.tsx:12:3)
```

### 新機能を実装する

```
「メールアドレスのバリデーション機能を追加して」
「APIのレスポンスをキャッシュする仕組みを実装して」
```

### コードをリファクタリングする

```
「この関数を読みやすくリファクタリングして」
「重複しているコードを共通化して」
```

### テストを書く

```
「この関数のユニットテストを書いて」
「カバレッジが低いファイルにテストを追加して」
```

---

## 精度を上げるコツ

### 具体的に伝える

| あいまいな依頼 | 具体的な依頼 |
|-------------|-------------|
| 「直して」 | 「`login` 関数のバリデーションエラーを修正して」 |
| 「追加して」 | 「ユーザー一覧ページに検索フィルターを追加して」 |
| 「きれいにして」 | 「`utils.ts` の関数を単一責任の原則に従ってリファクタリングして」 |

### コンテキストを共有する

関連するファイルや情報を一緒に伝えると精度が上がります：

```
「以下の仕様に従って実装して：
- ユーザーはメールとパスワードでログインできる
- ログイン失敗は3回まで。それ以降は10分ロック
- JWT でセッション管理する」
```

### 段階的に進める

大きなタスクは小さく分割して依頼すると安全です：

```
# NG: 一度に大きすぎるタスク
「ECサイト全体を作って」

# OK: 段階的に進める
1. 「商品一覧ページを作って」
2. 「商品詳細ページを追加して」
3. 「カートの追加/削除機能を実装して」
```

---

## ツール実行の承認

Claude Code はファイルの編集やコマンド実行の前に確認を求めることがあります。

- **承認する**: 変更内容を確認して問題なければ許可
- **拒否する**: 意図と違う場合は拒否して理由を伝える

```
「さっきの変更は意図と違った。〇〇の方法でやり直して」
```

---

## 会話のリセット

別のタスクに移る際は `/clear` で会話をリセットすると、
前の文脈が混入せずクリーンな状態で始められます。

```
/clear
```
