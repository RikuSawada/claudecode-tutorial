# CLAUDE.md の活用

---

## CLAUDE.md とは

プロジェクトルートに置く設定ファイルで、Claude Code に対して
**プロジェクト固有の情報・ルール** を伝えるためのものです。

Claude Code はセッション開始時にこのファイルを自動で読み込みます。
**毎回同じことを説明しなくて済む** ようになります。

```
your-project/
├── CLAUDE.md       ← ここに置く
├── src/
└── package.json
```

---

## 書くべき内容

| セクション | 内容例 |
|-----------|--------|
| プロジェクト概要 | 何を作るサービスか、技術スタック |
| 開発環境セットアップ | ローカルで動かすコマンド |
| アーキテクチャ | ディレクトリ構成、設計方針 |
| コーディング規約 | 命名規則、禁止パターン |
| 注意事項 | Claude に特に気をつけてほしい点 |
| 禁止事項 | やってはいけない操作 |

## 記入例

実際にどう書くかの参考例です。上のテンプレートをベースに、プロジェクト情報を埋めたものです：

```markdown
# CLAUDE.md

## Claude Code への注意事項

- 挨拶・前置き・段階報告・絵文字禁止。結論ファースト
- 指摘すべきことは率直に指摘
- 実装前に必ず該当ファイルを読んでから変更すること
- テストなしで実装を完了させないこと
- 破壊的な変更（DB スキーマ変更など）は事前に確認を取ること
- コミットは日本語でわかりやすく書くこと
- TODOには必ずブランチ作成・実装内容のテスト・コミット・push・PR作成が含まれるべきです

## プロジェクト概要

タスク管理 Web アプリ。ユーザーがタスクを作成・管理・共有できるサービス。
フロントエンドは HTMX によるサーバーサイドレンダリング、バックエンドは FastAPI で構成。

### Repository 設定
- **リポジトリ名**: RikuSawada/taskboard
- **Git remote**: git@github.com:RikuSawada/taskboard.git

### 技術スタック
- **フロントエンド**: HTMX + Jinja2 テンプレート
- **バックエンド**: FastAPI（Python 3.12）
- **DB**: PostgreSQL + SQLAlchemy（Async）+ Alembic（マイグレーション）
- **認証**: JWT（python-jose）
- **パッケージマネージャ**: uv

## 開発環境セットアップ

\`\`\`bash
uv sync
cp .env.example .env
uv run alembic upgrade head
uv run uvicorn app.main:app --reload
uv run pytest
\`\`\`

## アーキテクチャ

\`\`\`
taskboard/
├── app/
│   ├── main.py              # FastAPI エントリポイント
│   ├── routers/             # エンドポイント定義
│   ├── models/              # SQLAlchemy モデル
│   ├── schemas/             # Pydantic スキーマ
│   ├── crud/                # DB 操作ロジック
│   └── templates/           # Jinja2 テンプレート
├── alembic/                 # マイグレーション管理
└── tests/
\`\`\`

## コーディング規約

- **型ヒント**: すべての関数に引数・戻り値の型ヒントを付ける（`Any` 禁止）
- **命名**: 関数・変数は snake_case、クラスは PascalCase
- **テスト**: 新規エンドポイントには必ず対応するテストを追加する

## 注意事項

- `models/` を変更する前に必ずユーザーに確認を取ること
- マイグレーションは内容を確認してから `upgrade head` を実行する

## 禁止事項

- 型ヒントなしで関数を定義しない
- `Any` 型を使わない
- `.env` をコミットしない
- `git push --force` を `main` ブランチに対して実行しない
```

## グローバル設定との関係

| ファイル | 有効範囲 |
|---------|---------|
| `~/.claude/CLAUDE.md` | 全プロジェクト共通（個人の好みや作業スタイル） |
| `CLAUDE.md`（プロジェクトルート） | そのプロジェクト内のみ |

両方がある場合は、**プロジェクトの設定がグローバルに追加される形** で読み込まれます。

## CLAUDE.md を Git で管理する

チームで使う場合はコミットしておくと、メンバー全員が同じ設定で Claude Code を使えます：

```bash
git add CLAUDE.md
git commit -m "CLAUDE.md にプロジェクト情報を追記"
```

詳細は [Memory（記憶機能）](memory.md) も参照してください。
