# CLAUDE.md の活用

---

## CLAUDE.md とは

プロジェクトルートに置く設定ファイルで、Claude Code に対して
**プロジェクト固有の情報・ルール** を伝えるためのものです。

Claude Code はセッション開始時にこのファイルを自動で読み込みます。

```
your-project/
├── CLAUDE.md       ← ここに置く
├── src/
└── package.json
```

---

## 書くべき内容

### プロジェクト概要

```markdown
## プロジェクト概要

ECサイトのバックエンド API。
Node.js + Express + PostgreSQL で構成。
フロントエンドは別リポジトリ（frontend-repo）。
```

### よく使うコマンド

```markdown
## よく使うコマンド

# 開発サーバー起動
npm run dev

# テスト実行
npm test

# DB マイグレーション
npm run migrate
```

### アーキテクチャ・構成

```markdown
## ディレクトリ構成

src/
├── controllers/   # リクエストハンドラ
├── services/      # ビジネスロジック
├── models/        # DB モデル
└── routes/        # ルーティング定義
```

### コーディング規約

```markdown
## コーディング規約

- TypeScript を使用（型定義を必ず書く）
- 関数名はキャメルケース
- コンポーネント名はパスカルケース
- コメントは日本語で書く
- ESLint・Prettier のルールに従う
```

### Claude Code への指示

```markdown
## Claude Code への注意事項

- 実装前に必ず該当ファイルを読んでから変更すること
- テストなしで実装を完了させないこと
- 破壊的な変更（DB スキーマ変更など）は事前に確認を取ること
- コミットは日本語でわかりやすく書くこと
```

## 活用のポイント

- **短く具体的に書く**: 長すぎると読み飛ばされる可能性があります
- **「禁止事項」も書く**: 「〇〇はしないで」という制約も有効です
- **更新し続ける**: プロジェクトの成長に合わせて随時更新してください
- **チームで共有**: git でコミットしてチーム全員の Claude Code に反映させましょう
