# Skills（スキル）& スラッシュコマンド

---

## スラッシュコマンドとは

`/コマンド名` の形式で呼び出せる特殊な命令です。
よく使う操作をワンコマンドで実行できます。

---

## 組み込みスラッシュコマンド

| コマンド | 説明 |
|---------|------|
| `/help` | ヘルプを表示 |
| `/clear` | 会話履歴をリセット |
| `/compact` | 会話を要約して圧縮（長い会話で有効） |
| `/commit` | 変更を解析してコミットを実行 |
| `/review-pr [PR番号]` | Pull Request をレビュー |

---

## Skills（カスタムスキル）とは

Skills は `/スキル名` で呼び出せる **独自の定型タスク** です。
繰り返し使う指示をファイルに定義しておくことで、毎回書かずに済みます。

---

## カスタムスキルの作成方法

### 1. ファイルを作成する

`.claude/commands/` ディレクトリにMarkdownファイルを置きます。

```
your-project/
└── .claude/
    └── commands/
        ├── check-security.md
        ├── write-tests.md
        └── summarize-pr.md
```

### 2. スキルの内容を書く

ファイルの中身が、そのスキルを呼んだときに Claude に渡されるプロンプトになります。

例：`.claude/commands/check-security.md`

```markdown
以下の観点でコードをレビューして、問題点を日本語で報告してください：

1. OWASP Top 10 に基づく脆弱性
2. SQL インジェクション、XSS、CSRF のリスク
3. 認証・認可の不備
4. 機密情報のハードコード

問題が見つかった場合は、修正案も提示してください。
```

### 3. 呼び出す

```
/check-security
```

## スキルの活用例

### テスト自動生成

`.claude/commands/write-tests.md`：

```markdown
現在開いているファイルに対してユニットテストを書いてください。
- 正常系・異常系を網羅する
- エッジケースも考慮する
- 既存のテストフレームワークに合わせる
```

### PR サマリー作成

`.claude/commands/pr-summary.md`：

```markdown
このブランチの変更内容を以下の形式でまとめてください：

## 概要
（変更の目的を1〜2文で）

## 変更内容
- （箇条書きで主な変更点）

## テスト方法
（確認手順）
```

### コードレビュー

`.claude/commands/review.md`：

```markdown
以下の観点でこのコードをレビューしてください：
- 可読性・保守性
- パフォーマンス上の問題
- バグの可能性
- より良い実装方法の提案
```

## 引数を渡す

スキルに `$ARGUMENTS` を使うと、呼び出し時に引数を渡せます。

`.claude/commands/fix-issue.md`：

```markdown
GitHub Issue #$ARGUMENTS の内容を読み込んで、
修正を実装し、PRを作成してください。
```

呼び出し方：

```
/fix-issue 42
```
